# DevOps Mastery Journey – Git & GitHub Fundamentals

## Overview

This repository documents my hands-on learning journey through essential **Git and GitHub concepts used by DevOps Engineers**.
The goal of this section is to understand how version control works in real-world development environments and how teams collaborate using Git.

These notes are based on practical exercises, terminal commands, troubleshooting scenarios, and real DevOps workflows.

---

## Topics Covered

### Git Fundamentals

* What Git is and why version control is important
* Initializing repositories using `git init`
* Tracking changes using `git add`
* Saving changes using `git commit`
* Checking repository status using `git status`
* Viewing commit history using `git log`

### Working With Remote Repositories

* Connecting local repository to GitHub
* Uploading changes using `git push`
* Downloading projects using `git clone`
* Understanding HTTPS vs SSH authentication

### Branching and Collaboration

* Creating branches using `git checkout -b`
* Switching branches using `git checkout`
* Understanding why branches are important in team environments
* Merging branches using `git merge`
* Linear history with `git rebase`

### Conflict Handling

* Understanding merge conflicts
* Resolving conflicts manually
* Best practices when multiple developers modify the same file

### Security and Authentication

* GitHub Personal Access Tokens
* SSH authentication setup
* Generating SSH keys using `ssh-keygen`
* Using SSH agent for secure Git operations

## Practical Concepts Learned

This learning included solving real issues such as:

* SSH setup troubleshooting

Understanding these real scenarios is critical for working effectively in production DevOps environments.
