# AWS DevOps Core Concepts

This repository documents my learning journey through fundamental AWS and DevOps infrastructure concepts.
The goal of this repository is to build a strong conceptual understanding of how modern cloud systems are designed, secured, monitored, and scaled.

The notes focus on **core AWS services, container orchestration, monitoring systems, and security mechanisms** that are commonly used in real-world DevOps environments.

## Topics Covered

### Core AWS Infrastructure

* EC2 – Virtual servers in the cloud
* VPC – Virtual private network architecture
* EBS – Persistent block storage for EC2
* S3 – Object storage for scalable data storage

### Security and Access Control

* IAM – Identity and access management
* KMS – Encryption key management
* CloudTrail – AWS activity logging
* AWS Config – Resource configuration tracking

### Serverless and Compute

* AWS Lambda – Event-driven serverless compute

### Container and Orchestration

* ECS – AWS native container orchestration
* EKS – Kubernetes managed by AWS
* Fargate – Serverless container runtime

### DevOps and CI/CD

* CodeBuild – Cloud build service for compiling and testing code

### Monitoring and Logging

* ELK Stack – Centralized logging system
* Prometheus + Grafana – Metrics monitoring and visualization

### Cost and Governance

* AWS Billing and Cost Management

### Compliance and Security Standards

* ISO 27001
* HIPAA
* SOC2

## Purpose of This Repository

* Document my DevOps learning journey
